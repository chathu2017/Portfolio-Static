🚀 Secure Access from On-Premises to Azure

I implemented a hybrid networking project in Azure, enabling secure communication from an on-premises environment to Azure-hosted resources using IPSec (IKEv2) VPN tunneling...

I implemented a hybrid networking project in Azure, enabling secure communication from an on-premises environment to Azure-hosted resources using IPSec (IKEv2) VPN tunneling.


Architecture Overview:

  • 3 Azure VNets:
    • Hub VNet (192.168.0.0/24) – Hosted the Azure VPN Gateway
    • Spoke 1 VNet (192.168.1.0/24) – Deployed Nginx Web Server on a VM
    • Spoke 2 VNet (192.168.3.0/24) – Hosted Azure MySQL Flexible Server (Private Access only)
  • All VNets are peered, allowing seamless communication.
  • On-premises environment simulated using another Azure tenant + VNet with Windows Server 2022 configured as a router via RRAS.

🔐 The Result:

  • ✅ Successfully accessed Azure-hosted MySQL Flexible Server and Web App over private IP via a secure IPSec VPN tunnel.
  • ✅ MySQL connection is encrypted with SSL using Azure’s public CA certificate.
